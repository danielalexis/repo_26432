    Desenvolvimento da Sprint

    Nesta sprint foram efetuados os seguintes UC's:

    • UC1.5 - Realizar pagamento
    • UC2.4 - Gerenciar o check-in
    • UC2.5 - Gerenciar o check-out
    • UC2.6 - Receber pagamento
    • UC3.5 - Receber solicitações de serviços de excelência
    • UC3.6 - Programar e executar serviços de excelência
    • UC3.7 - Receber pagamento

    UC1.5 - Realizar Pagamento

    User Stories de Caso de Uso 1.5

    • US 1.5.1: Como Aluno, Eu quero realizar o pagamento por um serviço, Para concluir o processo de solicitação.

    Critérios para Aceitação do Caso de Uso 1.5

    • O sistema deve apresentar uma opção clara para o aluno realizar o pagamento.
    • O sistema deve fornecer um recibo detalhado após a conclusão bem-sucedida do pagamento.
    • Em caso de falha no pagamento, o sistema deve fornecer mensagens de erro claras e orientações para resolver a questão.

    UC2.4 - Gerenciar o Check-in

    User Stories de Caso de Uso 2.4

    • US 2.4.1: Como Proprietário, Eu quero gerenciar o check-in dos usuários, Para controlar o acesso às instalações.

    Critérios para Aceitação do Caso de Uso 2.4

    • O sistema deve permitir que o proprietário registe o check-in de usuários de maneira eficiente.
    • Durante o check-in, o proprietário deve validar a autenticidade das informações do usuário.
    • O proprietário deve ter a capacidade de visualizar o registo de check-ins anteriores.

    UC2.5 - Gerenciar o Check-out

    User Stories de Caso de Uso 2.5

    • US 2.5.1: Como Proprietário, Eu quero gerenciar o check-out dos usuários, Para manter um registro preciso da presença.

    Critérios para Aceitação do Caso de Uso 2.5

    • O sistema deve permitir que o proprietário registe o check-out de usuários de maneira eficiente.
    • O proprietário deve ter a capacidade de visualizar o registro de check-outs anteriores.
    • Em caso de tentativa de check-out inválida, o sistema deve notificar o proprietário.

    UC2.6 - Receber Pagamento

    User Stories de Caso de Uso 2.6

    • US 2.6.1: Como Proprietário, Eu quero receber pagamentos, Para facilitar transações financeiras.

    Critérios para Aceitação do Caso de Uso 2.6

    • O sistema deve fornecer uma interface segura para que o proprietário registre e processe pagamentos diretamente.
    • Durante o processo de pagamento, o sistema deve garantir a segurança das informações financeiras.
    • O proprietário deve receber uma notificação imediata após a conclusão bem-sucedida de um pagamento.
    • Em caso de falha no processamento do pagamento, o sistema deve fornecer mensagens de erro claras e orientações para resolver a questão.

    UC3.5 - Receber Solicitações de Serviços de Excelência

    User Stories de Caso de Uso 3.5

    • US 3.5.1: Como Técnico de Manutenção, Eu quero receber solicitações para serviços de excelência, Para oferecer um nível superior de atendimento.

    Critérios para Aceitação do Caso de Uso 3.5

    • O sistema deve permitir que os técnicos recebam notificações de solicitações de serviços de excelência.
    • Durante o atendimento, os técnicos devem seguir procedimentos específicos para oferecer um serviço excepcional.
    • Após a conclusão do serviço de excelência, o sistema deve permitir que os técnicos registrem detalhes sobre a experiência do cliente.

    UC3.6 - Programar e Executar Serviços de Excelência

    User Stories de Caso de Uso 3.6

    • US 3.6.1: Como Técnico de Manutenção, Eu quero programar e executar serviços de excelência, Para proporcionar uma experiência superior aos clientes.

    Critérios para Aceitação do Caso de Uso 3.6

    • O sistema deve permitir que os técnicos agendem serviços de excelência com base na disponibilidade e nas solicitações dos clientes.
    • Durante a execução do serviço de excelência, o técnico deve seguir um conjunto de procedimentos específicos para garantir um serviço excepcional.
    • Após a conclusão do serviço, o sistema deve permitir que os técnicos registrem detalhes sobre a execução bem-sucedida.

    UC3.7 - Receber Pagamento

    User Stories de Caso de Uso 3.7

    • US 3.7.1: Como Técnico de Manutenção, Eu quero receber pagamentos por serviços de excelência, Para concluir a transação.

    Critérios para Aceitação do Caso de Uso 3.7

    • O sistema deve fornecer uma interface para que os técnicos registrem e processem pagamentos por serviços de excelência.
    • Durante o processo de pagamento, o sistema deve garantir a segurança das informações financeiras.
    • Os técnicos devem receber uma notificação imediata após a conclusão bem-sucedida de um pagamento.
    • Em caso de falha no processamento do pagamento, o sistema deve fornecer mensagens de erro claras e orientações para resolver a questão.
