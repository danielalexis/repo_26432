    Desenvolvimento da Sprint

    Nesta sprint foram efetuados os seguintes UC's:

    • UC1.1 - Autenticação
    • UC1.2 - Fazer requisição
    • UC1.6 - Consultar requisição
    • UC2.1 - Autenticação
    • UC3.1 - Autenticação
    • UC3.2 - Receber solicitações de manutenção

    UC1.1 - Autenticação

    User Stories de Caso de Uso 1.1

    • US 1.1.1: Como Aluno, Eu quero autenticar-me no sistema, Para que eu possa aceder ao sistema.

    Critérios para Aceitação do Caso de Uso 1.1

    • O sistema deve fornecer uma interface de login para o aluno inserir suas credenciais.
    • Após inserir as credenciais, o sistema deve autenticar o aluno e conceder acesso ao sistema.
    • Se as credenciais fornecidas forem inválidas, o sistema deve informar ao aluno sobre a falha na autenticação.
    • Após a autenticação, o aluno deve ter acesso às funcionalidades específicas destinadas aos alunos.

    UC1.2 - Fazer Requisição

    User Stories de Caso de Uso 1.2

    • US 1.2.1: Como Aluno, Eu quero fazer uma requisição, Para solicitar uma residência.

    Critérios para aceitação do Caso de Uso 1.2

    • O sistema deve permitir que o aluno preencha um formulário de requisição.
    • A requisição deve incluir informações obrigatórias, como nome, instituição e data desejada.
    • Após submeter a requisição, o sistema deve gerar um número de referência único para acompanhamento.
    • Os alunos devem poder visualizar o histórico de suas requisições anteriores.

    UC1.6 - Consultar Requisição

    User Stories de Caso de Uso 1.6

    • US 1.6.1: Como Aluno, Eu quero consultar o status da minha requisição, Para acompanhar o progresso e obter informações atualizadas.

    Critérios para aceitação do Caso de Uso 1.6

    • Os alunos devem poder inserir o número de referência da requisição para consultar seu status.
    • O sistema deve exibir informações detalhadas sobre a requisição, como data de submissão, status atual e observações relevantes.
    • Caso a requisição tenha sido concluída, o sistema deve fornecer detalhes sobre a conclusão do serviço.

    UC2.1 - Autenticação

    User Stories de Caso de Uso 2.1

    • US 2.1.1: Como Proprietário, Eu quero autenticar-me no sistema, Para que eu possa gerir as solicitações dos alunos.

    Critérios para aceitação do Caso de Uso 2.1

    • O sistema deve validar as credenciais do proprietário durante o processo de autenticação.
    • A autenticação bem-sucedida deve proporcionar ao proprietário acesso às funcionalidades de gestão das requisições dos alunos.
    • Em caso de falha na autenticação, o sistema deve informar ao proprietário sobre a tentativa mal-sucedida.

    UC3.1 - Autenticação

    User Stories de Caso de Uso 3.1

    • US 3.1.1: Como Técnico de Manutenção, Eu quero autenticar-me no sistema, Para que eu possa visualizar e gerir as solicitações de manutenção.

    Critérios para aceitação do Caso de Uso 3.1

    • O sistema deve autenticar o técnico de manutenção por meio de credenciais específicas.
    • Após a autenticação, o técnico deve ter acesso às funcionalidades relacionadas à gestão de solicitações de manutenção.
    • Em caso de falha na autenticação, o sistema deve notificar o técnico sobre a tentativa malsucedida.

    UC3.2 - Receber Solicitações de Manutenção

    User Stories de Caso de Uso 3.2

    • US 3.2.1: Como Técnico de Manutenção, Eu quero receber novas solicitações de manutenção, Para que eu possa responder prontamente.

    Critérios para aceitação do Caso de Uso 3.2

    • O sistema deve enviar notificações imediatas ao técnico de manutenção quando uma nova solicitação for submetida.
    • As notificações devem incluir informações essenciais, como a descrição do problema e a localização.
    • O técnico deve poder aceitar ou recusar uma solicitação diretamente da notificação, facilitando uma resposta rápida.
